## 策略概述

当前生产代码仍使用历史 6 因子模型；无泄漏开发集只把 `momentum_20` 与 `volume_trend` 列为候选，尚未打开封存集验证。股票池为 6 大板块 49 只 A 股，持有期为 20 个交易日。

### 核心原则：因子集驱动，权重为辅

> 以下 v2.0→v2.6 分数轨迹是历史记录，已确认受前视偏差污染，不得作为当前能力证据。

v2.0→v2.6 的历史分数轨迹：

```
v2.0: 76.9 → v2.4: 79.7 (+2.8, 砍7死因子)
           → v2.5: 81.6 (+1.9, 加量价配合因子)
           → v2.6: 81.7 (+0.1, 加量能趋势因子)
```

**改变因子集涨分，调权重不涨。** 两次最大的跳升都来自结构性改动（砍因子/加因子）。所有参数性改动（指数加权、权重倍率、base-weight 替换）分数全部 ≤0。

### 两个量能因子的特殊价值

6 个因子中，4 个是价格维度的（momentum_20/60、reversal_5、max_drawdown），2 个是量能维度的（volume_corr、volume_trend）。量能因子和所有价格因子的相关性 r<0.25——它们提供的是纯增量信息，从"钱在往哪流"和"涨得是否健康"两个角度切入。

## 六个因子

每个因子的权重和存在理由都由 IC 分析决定，不是拍脑袋。

### momentum_20（权重 34%，IC = +0.084）

**是什么**：股票最近 20 个交易日的累计收益率。

**为什么有效**：A 股存在显著的中短期动量效应——过去一个月涨得好的股票，下个月倾向于继续涨。是所有因子中 IC 均值最高的。

**牛市/熊市调整**：牛市 ×1.5（追涨），熊市 ×0.3（减仓等企稳），震荡 ×0.8。

### momentum_60（权重 17%，IC = +0.055）

**是什么**：股票最近 60 个交易日的累计收益率。

**为什么有效**：捕捉中期趋势，和 momentum_20 的相关性仅 r≈0.48——它是独立信号。20 日动量告诉你"最近在涨"，60 日动量告诉你"已经涨了一阵子了，趋势更稳固"。

### max_drawdown（权重 16%，IC = -0.114）

**是什么**：股票在过去 60 个交易日内，从最高点到最低点的最大跌幅。取负值——回撤越小得分越高。

**为什么有效**：在 20 日窗口里，之前回撤小的股票确实表现更好。这也是比赛评分的直接对齐——回撤占 24 分。熊市 ×2.0（严格防守），牛市 ×0.7（适度放松）。

### reversal_5（权重 8%，IC = -0.094）

**是什么**：最近 5 个交易日的收益率，**取负值**。

**为什么取负值**：IC 为负意味着"跌了 5 天"的股票**继续跌**（短期动量而非反转）。翻转符号后成为独立的短期动量信号。权重最低——Std IC 0.299 是所有因子中最不稳定的。

### volume_corr（权重 19%，IC = +0.049）★ 稳定器

**是什么**：过去 20 天收益率与成交量的 Spearman 秩相关系数。

**为什么有效**：正值 = 量价同向（放量上涨/缩量下跌，趋势健康），负值 = 量价背离（缩量上涨/放量下跌，趋势异常）。**Std IC=0.094，5 因子中最低——信号虽弱但极稳定。** 和 momentum_20 的 r=0.644 说明它捕捉的是动量的"质量"维度而非动量本身。

**市态不变**：所有市场状态下权重保持不变（regime-invariant）。

### volume_trend（权重 6%，IC = +0.073）★ 微量锦上添花

**是什么**：过去 20 天成交量后 10 天均值 / 前 10 天均值 - 1。正值 = 量能放大。

**为什么有效**：成交量放大意味着资金关注度上升，价格可能后续跟上。IC 均值仅次于 momentum_20，且和所有价格因子正交（r<0.20）。但 Std IC=0.191，信号波动大，只能给微量权重。

### 波动率硬约束（不作为因子打分）

volatility 的 IC≈0，几乎无预测力，但它的角色不是预测收益而是**控制风险**。我们把它从打分公式里拿出来，改成硬门槛：**波动率 Z-score > 2.0 的股票直接排除**。这样高波股票被过滤（降低回撤），但不会因为"低波动加分"而过度集中在银行股。

## 市场判市

判市决定了因子权重的方向。当前用了三层判断：

### 第一层：技术面 MA 偏离

计算股票池等权均价相对 60 日均线的偏离，用 20 日波动率标准化。**标准化后"偏离几个标准差"在任何市场里含义一致**：日波 1.5% 的市场里 2σ=3%，日波 3% 的市场里 2σ=6%。

### 第二层：CSI 300 指数独立判市

个股等权均价里小盘噪声大。沪深 300 是市值加权的——更能代表"真正的市场"。同一套 MA + 标准化逻辑跑在指数上，做独立确认。

### 第三层：异常覆盖

当近期波动率飙升到历史的 2 倍以上，或收益率/波动率的比值深度为负时，不管 MA 怎么说，强制把牛市/熊市降级为震荡。防止高波噪声市场里的误判。

### 融合规则

两方一致 → 输出共识。一方牛一方震荡 → 偏向非震荡。一方牛一方熊 → 不可能，输出震荡保底。异常覆盖优先于一切。

## 选股流程

每 20 个交易日，执行以下步骤：

1. **判市** → 三层融合给出 bull/bear/range
2. **赋权** → 根据市态调整 6 个因子的权重倍率（量能因子市态不变）
3. **打分** → 每个因子在板块内 Z-score 标准化，加权得到 composite
4. **过滤** → 排除 vol_z > 2.0 的股票
5. **分散** → 每个板块至少选 1 只，最多 15 只，composite > 0 才纳入
6. **配仓** → 风险平价（逆波动率加权），单只 ≤10%，单板块 ≤25%
7. **回测** → 向量化回测计算收益率、回撤、Sharpe

### 空仓/轻仓规则

| 触发条件 | 动作 |
|----------|------|
| 超半数股票 composite < 0 | 仓位降至 30% |
| 市场均价 20 日跌幅 > 5% | 仓位降至 30% |
| 两者同时触发 | 100% 空仓 |

## 多 Agent 协作（v2.6 真多视角架构）

方向 8 验证后，Bull 和 Bear 使用**不同因子集**（不再同一套因子调权重）：

```
Coordinator (Quant PM)
  │  1. 获取数据 → 2. 判市 → 3. 因子适配评估
  │
  ├─→ Bull Analyst (趋势视角)
  │     quant_bull_view: momentum_20 + momentum_60 + volume_corr
  │     "哪些股票在涨且涨得健康？"
  │
  ├─→ Bear Analyst (风控视角)
  │     quant_bear_view: max_drawdown + reversal_5 + volume_corr(反向)
  │     "哪些股票可能突然跌？"
  │
  └─→ 双视角融合 → 共识优先 → 风控优先 → 最终组合 + 报告
```

**关键数据**：因子集分离后 Bull/Bear 选股 overlap 仅 28%，Spearman r≈-0.10。不再是同一套逻辑的三次重复——每个 Agent 有独立输入、独立视角、独立输出。

## 数据源

三层级联兜底，每层只补充上一层没拉到的股票：

```
akshare (东方财富)  →  快但可能被封
    ↓ 失败
baostock (自营服务器)  →  稳定，无频率限制，含基本面数据
    ↓ 失败
yfinance (Yahoo)  →  国际备用，国内慢
```

## 风险控制

| 约束 | 阈值 | 类型 |
|------|------|------|
| 单只股票 | ≤ 10% | 硬约束 |
| 单板块 | ≤ 25% | 硬约束 |
| 波动率排除 | vol_z > 2.0 | 硬约束 |
| 最低现金 | ≥ 5% | 软约束 |
| 回撤止损 | 组合回撤 > 15% | 动态触发（减半仓） |
| 空仓触发 | 个股面差 + 市场面差 | 动态触发 |
| 判市异常覆盖 | vol 飙升或收益/波动背离 | 牛市/熊市 → 震荡 |

## 技术栈

| 组件 | 选型 | 理由 |
|------|------|------|
| Agent 框架 | JiuwenSwarm | 比赛要求 |
| LLM | DeepSeek (deepseek-chat) | 性价比 |
| 数据源 | akshare + baostock + yfinance | 三层兜底 |
| 策略引擎 | 6 因子 IC 驱动 | 数据验证最优 |
| 因子验证 | IC (Spearman 秩相关) | 统计显著性 |
| 市场判市 | MA 波动率标准化 + 指数交叉验证 + 异常覆盖 | 三层融合 |
| 仓位管理 | 风险平价 | 低波多配、高波少配 |
| 回测引擎 | 向量化计算 | 纯 Python/numpy，零依赖 |
