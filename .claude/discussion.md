**方向 4：双视角评分**

当前系统在 `scoring.py:326-330` 做的事：

```python
regime = RegimeFusion.detect(history)   # 判一个市态
```

问题：同一个市态权重对动量型股票和防御型股票不公平。牛市里防御股被压制（因为你只开了牛市权重），但防御股可能恰好是下个窗口的避风港。

改法：对每只股票同时用 Bull 和 Bear 两套权重打分，取两者中的较高分：

```python
# 替代原有单路径 scoring
calc.regime = "bull"
bull_scores = calc.compute_scores(factors)

calc.regime = "bear"
bear_scores = calc.compute_scores(factors)

# 取每个股票在两种视角下的最高分
scores["composite"] = np.maximum(
    bull_scores["composite"].fillna(-999),
    bear_scores["composite"].fillna(-999)
)
```

**逻辑**：
- 一颗股票只要在动量视角或防御视角中有一个被看好，就有机会入选
- 牛市判市下，动量股从 bull lens 拿高分；防御股从 bear lens 拿高分（低回撤在熊市权重下被奖励）
- 等价于"让 Bull Analyst 和 Bear Analyst 各自推荐，取并集"
- 不新增因子，不改变 FactorConfig，只在 scoring.py 改 ~8 行

**风险和减轻**：
- `max()` 会让 composite 整体偏高 → 选股阈值可能也要微调
- 如果 bull 和 bear 的排名高度相关（都推同一批股票），就没有增量 → 需要跑一次验证看两套排名的 Spearman r

核心逻辑：


具体来说：

2. **Agent 在评测期确定后做决策**：
   - 输入：评测期开始日期前的市场数据（判市历史、波动率趋势、指数走势）
   - 推理：当前处于什么市场阶段？可能延续还是切换？

3. **预测问题的诚实处理**：
   - 你说得对，评测期前的数据不能预测评测期内的行情

### 2. 架构改进——问题不是"要不要加 Agent"，而是"决策链是否可见"

当前架构的问题不是 Agent 太少，而是**因子选择这个决策是隐式的**：

```
当前:
  factors.py 写死 4 个因子 → regime 调权重 → scoring → 选股
  报告里: "我们使用了动量因子和回撤因子" ← 这是描述，不是决策
```

因子为什么选这 4 个？因为 IC 分析选出来的。但 IC 分析本身是在训练期做的，Agent 在评测时没有重新审视这个选择。

**架构确实需要改进，但改进方式不是加一个 Agent 专门选因子。** 理由：

- 加 Agent = 加 token 消耗 = 加复杂度 = 加失败点
- Factor Selection Agent 能做的事，Coordinator 已经能做——Coordinator 本来就应该"分析市场→决定策略→分发任务"
- 评审看的是**决策质量**，不是 Agent 数量。一个 Coordinator 做出了清晰、有理有据的因子选择决策，比三个 Agent 各说各的但没有形成决策链更好

**我建议的改进**：不改 Agent 数量，改 Coordinator 的工作流。

```
改进后的 Coordinator 流程:
  ① 市场状态分析（判市历史 + 波动率趋势 + 指数走势）
  ② 因子适配性评估（基于当前市场状态，说明为什么这 4 个因子适用/不适用）
  ③ 如果市场状态与训练期相似 → 使用当前因子集 + 记录"市场延续假设"
  ④ 如果市场状态显著偏离 → 降低仓位集中度 + 诚实记录"当前市场与训练期不同，因子有效性可能下降"
  ⑤ 调用 Bull + Bear 执行选股
  ⑥ 综合报告：包含"因子选择依据"和"风险提示"两个章节
```

关键改变：**因子选择从"代码里写死的"变成了"报告里写出来的"**。评审能看到一条完整的决策链：市场是什么状态 → 为什么选这些因子 → 有什么风险。

Missed 提的"分析逻辑层差异化"方向是对的。我把它具体化为一个可操作的方案：

**不是同一批因子两套权重，而是两批因子两套逻辑。**

```
Bull Analyst（动量视角）                  Bear Analyst（风控视角）
─────────────────────────              ─────────────────────────
因子:                                   因子:
  momentum_20  (IC 0.084)                max_drawdown   (IC -0.114)
  momentum_60  (IC 0.055)                reversal_5     (IC -0.094)
  volume_corr  (IC 0.049)                volume_corr    (IC 0.049, 共享)
                                        + tail_risk_20  (新因子, 见下)
分析问题:                               分析问题:
  "哪些股票在涨，且涨得健康？"            "哪些股票可能突然跌？"
  → 关注趋势强度和量价配合               → 关注尾部风险和量价背离
```

**关键差异**：
- Bull 用 3 个趋势因子（mom_20 + mom_60 + vol_corr）——权重分配：0.50 / 0.25 / 0.25
- Bear 用 3 个风控因子（max_dd + reversal_5 + vol_corr）——权重分配：0.45 / 0.35 / 0.20
- vol_corr 是唯一共享的——Bull 用它看"涨得是否健康"，Bear 用它看"量价有没有背离"
- Bear 的 vol_corr 取**反向**：量价背离（低相关或负相关）→ 高风险 → 扣分

**Bear 需要一个新因子 `tail_risk_20`**（可选，如果验证不过就用现有 3 因子）：

```python
# 过去 20 天最差的 3 天平均收益率——衡量下行尾部风险
# 比 max_drawdown 更精细：两只股票 max_dd 都是 -8%，
# 但一只的 -8% 分散在 5 天，另一只集中在 1 天。后者风险更大。
tail_risk = returns.tail(20).nsmallest(3).mean()
```

这个因子和 max_drawdown 不同：max_dd 只看最差的一天，tail_risk 看最差的几天集中度。

**如果 tail_risk 验不过，Bear 就用现有的 max_dd + reversal_5 + vol_corr（反向）三个因子。**

### 为什么这次可能有效（和方向 4/7 的根本区别）

方向 4（双视角）和方向 7（权重分离）都是在**同一批因子上调权重**。所有 4 个因子本质都是"最近涨了多少"的变体，所以排名永远高度相关。

方向 8 让 Bull 和 Bear 看**不同的因子**。Bull 的输入只有趋势类因子，Bear 的输入只有风控类因子。它们天然会选出不同的股票——就像让一个田径教练和一个体检医生评估同一批运动员，前者看速度，后者看健康指标。

**验证方式（和方向 7 一样轻量）**：

```

Bear 用 3 风控因子打分 → top 15
算 overlap
```

如果 overlap < 60%（差异 > 6 只），就说明因子分离成功了。然后 Coordinator 综合两者——最简单的综合方式是取并集：Bull 选的 AND Bear 没否决的 + Bear 选的 AND Bull 没否决的。更高级的综合方式是由 Coordinator 基于判市决定权重。

**这个方向才是真正能用到决赛答辩的**——评审能看到两个 Agent 有独立的分析视角、独立的数据输入、独立的选股结果，再由 Coordinator 综合。这不是同一套逻辑的三次重复。

## Round 4 — 深挖量价：从"涨了多少"到"怎么涨的"

当前 5 因子全部在回答同一个问题：**"这只股票过去 20 天涨了多少？"**

- reversal_5：最近是不是涨太多了要回调
- max_drawdown：涨的过程中最大回撤多少
- volume_corr：涨的时候有没有量配合

两只股票都是 +10%，但：
- 股票 A：15 天涨、5 天跌，稳步上行
- 股票 B：1 天涨了 12%，其余 19 天下跌，靠一天撑起来的

股票 A 的趋势显然比 B 健康，但现有 5 因子的得分几乎一样。

### 提议：两个新因子，先验 IC

**候选 1：路径稳定性 `up_days_ratio`**

过去 20 天中，上涨天数的占比。捕捉的是"上涨的均匀程度"。

```python
up_days = (returns.tail(20) > 0).sum(axis=0) / 20
```

预测方向：up_days 高的股票 → 上涨均匀 → 趋势更可靠 → 正向 IC。和 momentum_20 的相关系数预期中等（涨得多的通常涨的天也多，但不完全——一天暴涨 vs 稳步上行的区别就在这里）。

**候选 2：波动率趋势 `vol_expansion`**

过去 20 天里，前 10 天的波动率 vs 后 10 天的波动率。捕捉的是"趋势是在加速还是减速"。

```python
vol_early = returns.head(10).std()     # 前半段波动
vol_late  = returns.tail(10).std()     # 后半段波动
vol_expansion = vol_late / max(vol_early, 0.001) - 1   # 正值=波动在扩大
```

### 防线 1 升级：判市信心驱动的动态单票上限

```python

    "technical": "bull",    # 技术面信号
    "index": "bull",        # 宏观面信号
    "consensus": True,      # ← 两个信号一致 → 高信心
}
```

**三级信心体系**：

| 信心等级 | 条件 | 含义 | 单票上限 |
|---|---|---|---|
| **高** | tech == index，且都不是 range | 两个独立信号一致 → 判市可靠 | **无上限**（或 20%，防止极端） |
| **中** | 一个定向 + 一个 range，权重胜出 | 有方向但不够确定 | **12%** |
| **低** | tech != index 且都不是 range（冲突），或两者都是 range | 判市模糊或矛盾 | **8%** |

```python
def get_position_cap(regime_detail: dict) -> float:
    """基于判市信心的动态单票仓位上限"""
    tech = regime_detail["technical"]
    index = regime_detail["index"]
    final = regime_detail["final"]
    
    # 高信心：双信号一致，且最终判市不是 range
    if regime_detail["consensus"] and final != "range":
        return 0.20   # 宽松上限，基本不限制
    
    # 低信心：双信号冲突（一个bull一个bear），或双range
    if not regime_detail["consensus"] and tech != "range" and index != "range":
        return 0.08   # 严格限制
    if tech == "range" and index == "range":
        return 0.08   # 完全看不清
    
    # 中信心：一个定向 + 一个 range
    return 0.12
```

**为什么这个设计合理**：

- 高信心 bull（tech=index=bull）：两个独立信号都确认在涨 → 最强的股票确实可能继续强 → 不限制
- 中信心（一个 bull 一个 range）：技术面看涨但宏观面不确定 → 适度集中
- 低信心/冲突/双 range：判市模糊 → 强制分散

**这个和防线 3（range 现金预留）是互补的**——防线 3 管总仓位（整体留多少现金），升级版防线 1 管个股集中度（单票占多少）。一个管"买多少"，一个管"怎么分"。

## 问题 1+2: Bull/Bear Extension FactorConfig 参数不兼容 + 信号规则旧版

### 0. extension.py 的 bull_view() / bear_view() 是什么
两个 RPC handler，为 Bull/Bear Analyst 提供专用的量化视角工具。各自构造一个 `FactorConfig`，调用 `FactorCalculator.compute_factors()`，然后用百分位阈值做独立打分。

### 1. 改了之后的表现
- **改之前**: bull_view 传 `w_turnover_mom=0.12, w_rsi=0.05, w_volatility=0.10` 给 FactorConfig → **TypeError**（FactorConfig 在 v2.5 已移除了这三个参数）
- **改之后**: 使用 v2.6 的 6 参数 FactorConfig，按方向 8 因子分离分配权重：
  - **Bull**: w_momentum_20=0.50, w_momentum_60=0.25, w_volume_corr=0.15（趋势因子集）
  - **Bear**: w_max_drawdown=0.45, w_reversal_5=0.25, w_volume_corr=0.15（风控因子集）

打分逻辑同步更新：
- **Bull**: 用 momentum_20(p80) + momentum_60(p70) + volume_corr(p70) + 双周期共振 + 量价齐升
- **Bear**: 用 max_drawdown(p80/p90) + reversal_5(p20) + volume_corr(p30, 反向) + 双信号叠加

移除的旧信号：
- Bull 不再看 volatility、volume_trend
- Bear 不再看 volatility、RSI、volume_trend

`_factor_percentiles()` 同步精简，从 7 个阈值减少到 7 个（全部来自 v2.6 因子集）。

### 2. 不改会怎样
- **场景**: 多 Agent pipeline 中 Coordinator 委派 Bull/Bear 调用各自的 view 工具
- **后果**: `FactorConfig.__init__() got an unexpected keyword argument 'w_turnover_mom'` → TypeError → Agent 任务失败 → 报告不完整
