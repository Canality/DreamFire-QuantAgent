开发要求
必须基于openJiuwen开源社区的相关代码仓，深度集成其核心能力模块；
作品基于Apache 2.0开源协议，赛后开源至openJiuwen社区;
有效作品
参赛Agent必须基于openJiuwen社区开源代码开发，必须修改JiuwenSwarm源码，否则视为无效作品。

提交格式
所有作答文件汇总后打包成zip压缩包，以团队名称命名压缩包（如：团队名称.zip），团队每日最多可提交3次，以参赛团队每个提交周期最后一次提交为准进行评测。

作品具体要求：
Agent 完整代码
包含所有业务代码、配置文件、依赖项声明（如 requirements.txt 或 environment.yaml），确保可一键部署。
量化投资报告（PDF、Word、Markdown等格式）
需涵盖选股逻辑、仓位决策依据、分析过程、风险评估以及最终投资组合明细。
投资组合结果
按组委会规定的格式列出最终选定的股票代码及其持仓占比，文件需与报告所述完全一致。
资源消耗日志
包括但不限于各阶段 Token 用量统计、端到端运行时长、峰值 GPU/CPU 利用率及显存消耗快照。
可复现说明文档（README）
清晰描述运行环境配置、执行步骤、关键参数释义以及可能遇到的问题与注意事项。
框架优化说明（如有）
若对 openJiuwen 框架本身进行了改进或扩展，应提交相应的补丁或分支代码，并附上详细的修改说明与效果对比。
解题思路推荐
为帮助参赛队伍快速启动开发工作并持续提升 Agent 的生成质量与效率，提供以下基于 openJiuwen 内置能力的快速搭建，供参考选型与迭代优化。
以赛事官方提供的基线工具包为基础，围绕 jiuwenswarm 的核心技能（Skills）系统 和 团队技能系统构建 Agent。建议路径如下：

单 Skill 流水线：将量化投资报告生成拆解为数据获取、因子计算、选股决策、报告撰写等环节，分别封装为独立 Skill，通过任务编排器串联执行。可直接复用框架内置的数据连接器与文档生成 Skill，最大程度降低开发门槛，快速产出可运行的基线方案。
Swarm Skill 协作：在单 Skill 基础上，将分析任务进一步分解为多个子 Skill（如技术分析 Skill、基本面分析 Skill、风险评估 Skill），并由一个顶层编排 Skill 负责调度与结果融合。该模式天然契合 MCP 通信协议，便于后续扩展为多 Agent 协同架构。

基线方案的优势在于开发成本低、可复现性强，适合快速熟悉 openJiuwen 框架并验证核心业务逻辑。
可参考市面上此类Agent的部分方法实现，参考链接:https://github.com/TauricResearch/TradingAgents
以上思路供各参赛队伍结合自身技术特长与时间安排灵活选用，鼓励在 openJiuwen 框架允许的范围内进行最大程度的创新与突破。
