# 02 — 回测框架：怎么判断策略是真的好还是在过拟合

**写给有 ML 背景的人**：回测（Backtest）就是金融领域的模型评估。但比 Kaggle 的 validation 更危险——金融数据的时间序列结构让过拟合更隐蔽，后果也更严重。

---

## 1. 回测是什么

> 回测 = 用历史数据模拟策略在过去的表现，以此推断未来

你在 Kaggle 里用 train data 训练模型，用 validation data 评估。量化里完全一样——用历史数据跑策略，看它在"过去"表现如何。

但问题来了：**金融市场只有一个 history，你不能做 k-fold CV。**

时间序列固定，你不能把 2024 年和 2026 年的数据 shuffle 后交叉验证——因为 2026 年的事情不可能发生在 2024 年之前。

---

## 2. 一段回测代码拆解

我们用的回测引擎核心逻辑：

```python
# 1. 计算每日收益率
daily_returns = prices.pct_change()

# 2. 按权重计算组合收益
portfolio_return = sum(weight_i × return_i for each stock)

# 3. 扣除交易成本
portfolio_return[0] -= transaction_cost × sum(weights)

# 4. 累积净值曲线
nav = (1 + portfolio_return).cumprod()

# 5. 计算指标
total_return = nav[-1] / nav[0] - 1
max_drawdown = min((nav - nav.cummax()) / nav.cummax())
sharpe = annualized_return / annualized_volatility
```

### 跟 Kaggle 代码的对应关系

| 回测代码 | Kaggle 代码 | 说明 |
|----------|------------|------|
| `prices.pct_change()` | `df.groupby('id')['value'].pct_change()` | 计算收益率 |
| `sum(w_i × r_i)` | `model.predict(X_test)` | 策略输出 |
| `nav.cumprod()` | 累积 log_loss | 累积表现 |
| `max_drawdown` | `max(train_score - val_score)` | 最差回撤 |
| `sharpe` | `accuracy / std(accuracy)` | 风险调整后收益 |

## 3. 回测的五个关键指标

我们的回测输出了这些数字：

```
Total Return:     +18.87%
Annualized Return: +40.55%
Max Drawdown:      4.77%
Sharpe Ratio:      5.46
Volatility (ann.): 7.42%
Win Rate:          60.9%
```

每个指标告诉你什么？

### 3.1 总收益率（Total Return）

$$ R_{total} = \frac{V_{end} - V_{start}}{V_{start}} $$

最直观的数字。100 万变 118.87 万，赚了 18.87%。

**ML 类比**：Test accuracy = 85%。但只看 accuracy 不够——要看混淆矩阵、F1、AUC 等。

### 3.2 年化收益率（Annualized Return）

$$ R_{ann} = (1 + R_{total})^{252/n} - 1 $$

其中 252 是 A 股一年的交易日数，n 是回测天数。

为什么要年化？因为不同回测周期长短不同：6 个月 +18.87% 和 3 个月 +12%，哪个更好？年化后统一到"一年"的口径：+40.55% vs +57.3%，后者更好。

**ML 类比**：相当于把不同数据集上的 accuracy 标准化到同一个 baseline 上比较。

### 3.3 最大回撤（Max Drawdown）

$$ MDD = \max_{t} \frac{P_{peak}(t) - P(t)}{P_{peak}(t)} $$

**这是最重要的风险指标，没有之一。**

你在 Kaggle 里关心模型的 worst-case performance（比如某个 fold 的 CV 特别差）。最大回撤告诉你：**最坏情况下，你的钱会从最高点亏多少。**

4.77% 的回撤意味着：如果你在最高点投了 100 万，最惨的时候账上只剩 95.23 万。

**为什么比赛评分里回撤占 30%？** 因为一只回撤 50% 的基金——即使最终赚了 30%——投资者在腰斩的时候就全部赎回了，根本等不到反弹。

### 3.4 Sharpe 比率

$$ Sharpe = \frac{R_{ann} - R_f}{\sigma_{ann}} \approx \frac{R_{ann}}{\sigma_{ann}} $$

（$R_f$ 是无风险利率，A 股通常忽略，或用 2-3%）

**Sharpe 衡量的是"性价比"**：每承担 1% 的波动风险，换来多少超额收益。

我们的 5.46 是用模拟数据跑出来的，真实 A 股市场：
- Sharpe > 2.0 = 顶尖水平
- Sharpe 1.0-2.0 = 优秀策略
- Sharpe 0.5-1.0 = 可接受
- Sharpe < 0.5 = 不如买指数基金

**ML 类比**：相当于 `(val_accuracy - baseline_accuracy) / std(val_accuracy)`。光 accuracy 高没用——如果每个 fold 的 accuracy 差别巨大（方差大），这个模型你敢上线吗？

### 3.5 胜率（Win Rate）

$$ WinRate = \frac{n_{positive\_days}}{n_{total\_days}} $$

60.9% 胜率意味着每 10 个交易日里，约 6 天在赚钱。

**但要注意**：胜率 90% 的策略可能每次赚 0.1%，亏一次亏 10%——胜率单独看没有意义，必须配合**盈亏比**一起看。这是你 Kaggle 经验里 precision vs recall 的 trade-off 在金融里的翻版。

## 4. 回测过拟合：金融里的 Data Leakage

这是最重要的一节。你在 Kaggle 里犯过的每一个过拟合错误，在量化里都有对应的坑——而且因为时间序列的特殊性，坑更深。

### 4.1 Look-Ahead Bias（未来信息泄露）

最常见的错误：用今天才知道的信息，去预测今天。

**错误示例**：
```python
# 用全年数据的均值做标准化，然后回测1月
scaler = StandardScaler()
scaled = scaler.fit_transform(all_data)  # 用了12月的数据！
january_data = scaled[:31]  # 1月已经"知道"了全年信息
```

**正确做法**：
```python
# 在 1 月底，你只能用 1 月及之前的数据
scaler = StandardScaler()
scaler.fit(data[:31])  # 只用1月的数据fit
january_scaled = scaler.transform(data[:31])  # transform 1月
```

这就是金融里的"expanding window"——你只能使用当前时间点之前的信息。

### 4.2 Survivorship Bias（幸存者偏差）

我们的比赛股票池是 49 只**现在还在交易的股票**。但如果在回测期间某只股票已经退市了呢？如果你仍然把它包含在内，回测结果就被高估了。

比赛因为是固定股票池，这个问题不大。但自己学习时要注意。

### 4.3 Over-Optimization（过度优化）

如果你有 20 个因子，你来回试不同的权重组合，挑出回测表现最好的那个——这就是 Data Leakage。

你在 Kaggle 里一定做过：反复提交看 LB score，挑表现最好的参数组合。这在 Kaggle 里叫"LB overfitting"，在量化里叫"回测过拟合"。

**防御方法**：
- **样本外测试**：留一段数据完全不看不碰，只在最后用一次
- **交叉时间段验证**：在 2023、2024、2025 三个不同年份分别回测，看表现是否一致
- **敏感性分析**：把因子权重上下浮动 10%，结果是否剧烈变化？如果微调权重导致结果大变 → 过拟合

## 5. 回测 vs 实盘：为什么会有差距？

这是量化交易最痛苦的真相：

| 你在回测里看到的 | 实盘中会发生 |
|-----------------|-------------|
| 收盘价成交 | 实际成交价可能偏离 0.1-0.5% |
| 忽略的滑点 | 大单买入会把价格推上去 |
| 完美的流动性 | 你想卖的时候可能没人接 |
| 0.03% 交易成本 | 实际可能 0.05-0.1% |

**ML 精确类比**：这就是你在 Kaggle 里 LB 排名很高，但实际部署后效果打折的原因——训练环境和真实环境的分布偏移（distribution shift）。

## 6. 你为什么看到"模拟数据"

今天跑回测时 akshare 因为网络限制连不上，我们用 numpy 生成了模拟数据。

```python
np.random.seed(42)
daily_returns = np.random.normal(mu, sigma, n)  # 正态分布日收益
price = base_price * np.cumprod(1 + daily_returns)
```

这模拟了"理想世界"——日收益服从正态分布、没有肥尾、市场不会崩盘。所以 Sharpe 5.46 完全不可信。

**真实金融数据长这样**：
- 日收益不服从正态分布（有"肥尾") 
- 隔三差五来一次暴跌（-5% 以上的日子比你想象的多）
- 波动率聚集（volatility clustering：大跌之后跟着大跌，大涨之后跟着大涨）

等接上真实数据后，我们的 Sharpe 大概率会从 5+ 掉到 1-2 之间。

## 7. 本节关键概念速查

| 概念 | 一句话 | ML 近似 |
|------|--------|---------|
| 回测 | 用历史数据模拟策略表现 | Train/val evaluation |
| 最大回撤 | 从峰值到谷底的最大跌幅 | Worst fold performance |
| Sharpe | 单位风险的超额收益 | Accuracy / std(accuracy) |
| 年化 | 统一到"一年"的标准 | Normalizing metrics across datasets |
| Look-ahead bias | 用了未来的信息 | Data leakage |
| 样本外测试 | 留一段数据完全不碰 | Hold-out validation |
| 肥尾 | 极端事件比正态分布预测的多 | Outliers that matter more than you think |
