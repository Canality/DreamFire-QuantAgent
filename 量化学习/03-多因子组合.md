# 03 — 多因子组合：怎么把多个信号合成一个决策

**写给有 ML 背景的人**：单因子 = 单特征模型。多因子 = Ensemble。因子合成 = 你决定每个特征给多少权重。

---

## 1. 从单因子到多因子

单因子策略：选 20 日动量最高的 10 只股票，每月调仓。

问题：2024 年 2 月 A 股暴跌时，高动量股票跌得最惨——动量因子在趋势反转时就是灾难。

多因子策略：同时看动量 + 波动率 + 回撤 + 成交量，综合打分。

**ML 类比**：单因子 = 贪心算法，只看一个 feature。多因子 = 你同时考虑几十个特征，避免被噪声主导。

---

## 2. IC：衡量因子好不好的核心指标

### IC 的定义

$$ IC_t = corr(FactorScore_{t-1}, Return_t) $$

翻译：在第 t 天，因子上个月的分数和这个月的实际收益率之间的相关系数。

- IC > 0：因子分数高的股票确实涨了 → 因子有效
- IC ≈ 0：因子分数和涨跌没关系 → 因子无效
- IC < 0：因子分数高的反而跌了 → 因子反向（取负号就能用）

**ML 直接类比**：

```python
# 你在 Kaggle 里做 feature importance:
from sklearn.feature_selection import mutual_info_regression
mi = mutual_info_regression(X, y)

# 金融里的 IC 就是:
ic = np.corrcoef(factor_scores, future_returns)[0, 1]
```

区别：feature importance 是全局的，IC 是每期都在变化的——一个因子可能这个月好用，下个月失效。

### IC 的三个衍生指标

| 指标 | 公式 | 含义 |
|------|------|------|
| **IC Mean** | mean(IC) | 因子长期平均有效性 |
| **IC Std** | std(IC) | 因子稳定程度 |
| **IR** | IC Mean / IC Std | 信息比率：综合衡量"有效且稳定" |

**IR 是最关键的因子评价指标**。IR > 0.5 就是好因子，IR > 1.0 是顶级因子。

类比：
- IC Mean 高 + IC Std 高 = 模型 accuracy 高但方差大 → 不稳定
- IC Mean 中 + IC Std 低 = 模型 accuracy 一般但每个 fold 都差不多 → 可信赖

**金融行业偏好后者**。稳定中等比忽高忽低好——因为你可以通过加杠杆放大稳定收益，但没法消除不稳定性。

## 3. 因子合成：怎么把多个因子变成最终得分

我们今天用的方法：

### 步骤 1：板块内 Z-score 标准化

```python
for sector in all_sectors:
    sector_data = raw_factors[raw_factors.sector == sector]
    z_scores = (sector_data - sector_data.mean()) / sector_data.std()
```

### 步骤 2：加权求和

```python
composite_score = (
    w1 * momentum_20_z +
    w2 * momentum_60_z +
    w3 * (-volatility_z) +      # 取负：低波动 = 高分
    w4 * (-max_drawdown_z) +     # 取负：低回撤 = 高分
    w5 * volume_trend_z
)
```

### 步骤 3：排序选股

```python
top_stocks = composite_score.sort_values(ascending=False).head(15)
```

## 4. 因子权重怎么定？三种方法

| 方法 | 做法 | 优点 | 缺点 |
|------|------|------|------|
| **等权** | 所有因子权重一样 | 最简单，不易过拟合 | 忽略因子质量差异 |
| **IC 加权** | 根据历史 IC 分配权重 | 数据驱动，自适应 | 可能过拟合历史 |
| **经验加权** | 基于经济学逻辑手动设定 | 可解释性强，稳健 | 依赖主观判断 |

我们今天用的是**经验加权**——给风险因子更高权重是因为比赛评分里回撤占 30%。

这不是最优做法，但作为 baseline 很合理。后续迭代可以尝试 IC 加权。

**ML 类比**：

| 方法 | ML 对应 |
|------|---------|
| 等权 | Bagging（所有模型同权重） |
| IC 加权 | Boosting（根据表现调整权重） |
| 经验加权 | Domain knowledge 手动设计特征 |

## 5. 多空组合：量化的核心思想

你不一定需要知道哪只股票会涨。**你只需要知道哪只比哪只好。**

假设你买入了因子分最高的 20%（多头），卖空了因子分最低的 20%（空头）：

- 如果 top 组涨了 5%，bottom 组跌了 3% → 你赚 8%
- 如果市场整体跌了 10%，top 组跌了 5%，bottom 组跌了 15% → 你还是赚 10%

多空组合剥离了市场涨跌（Beta），只保留因子能力（Alpha）。

**ML 类比**：相当于 paired testing——你关心的不是绝对预测准确率，而是 A 模型是否显著好于 B 模型。

我们的比赛策略只做多头（long-only），不做空——因为比赛评分基于纯多头持仓的绝对收益和回撤。但"多空的思维"在因子设计阶段很重要：你的因子应该能区分"谁好谁坏"。

## 6. 本节关键概念速查

| 概念 | 一句话 | ML 近似 |
|------|--------|---------|
| IC | 因子值与未来收益的相关系数 | Feature importance |
| IR | IC 的均值除以标准差 | Feature stability score |
| 因子合成 | 加权求和多个标准化后的因子值 | Ensemble prediction |
| 板块中性化 | 板块内 Z-score，消除板块偏差 | Group normalization |
| 多空组合 | 买好的、卖差的，剥离市场 Beta | Paired comparison |
| Alpha | 策略独有的超额收益 | Model's unique predictive power |
| Beta | 市场给你的收益（随大流的部分） | Baseline model's performance |
