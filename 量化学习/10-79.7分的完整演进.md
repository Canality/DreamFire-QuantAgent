# 10 — 79.7 分的完整演进：从 76.9 到接近满分

## 先看结果

```
7/16  v2.0:  76.9 ──── 起点（多Agent重构完成）
7/17  v2.1:  78.5 ──↑ +1.6  修了4个工程bug
      v2.2:  78.5 ─── 持平   判市改进（效果被bug抵消）
      v2.3:  77.2 ──↓ -1.3  加了PE/PB/ROE（反而退步）
      v2.4:  79.7 ──↑ +2.5  砍掉7个死因子，精简化
```

净增 +2.8 分。其中 +2.5 来自"砍掉不该加的因子"，+0.3 来自修 bug 和判市优化。

## 核心教训：在量化里，少就是多

这是我们今天学到的最大一条经验。

v2.3 的时候我们把因子从 8 个加到 11 个（加了 PE、PB、ROE 三个基本面因子），觉得"更全面了"。结果分数从 78.5 掉到 77.2。

然后我们用了一个叫 **IC（Information Coefficient）分析**的方法，逐个检验每个因子对 20 日收益的预测力：

| 因子 | IC | 含义 |
|------|-----|------|
| momentum_20 | +0.72 | 涨得多的真的会继续涨 |
| turnover_momentum | +0.69 | 但和 momentum_20 相关系数 0.92——几乎同一个信号 |
| RSI | +0.58 | 有效，但和 momentum_20 相关系数 0.75——冗余 |
| momentum_60 | +0.41 | 独立信号（和 mom_20 的 r=0.48） |
| reversal_5 | -0.39 | **方向反了！** 我们以为是反转信号，实际是动量延续 |
| max_drawdown | -0.38 | 跌得少的真的表现更好 |
| PE | -0.01 | 和抛硬币一样 |
| PB | -0.07 | 同上 |
| ROE | +0.08 | 微弱且不稳定 |
| volume_trend | 0.00 | 完全没用 |

基于这个数据，我们：
1. **砍掉 PE/PB/ROE/volume_trend**（IC ≈ 0，纯噪声）
2. **砍掉 turnover_momentum 和 RSI**（和 momentum_20 高度重复）
3. **翻转 reversal_5 的符号**（从"反转"改成"短期动量"，和它实际预测的方向一致）
4. **把 volatility 从因子改成硬约束**（"波动率超 2σ 的股票直接排除"，而不是在打分里给它权重）

从 11 个因子减到 4 个：mom_20、mom_60、reversal_5（翻转为 5 日动量）、max_drawdown。**分数反而创新高 79.7。**

## 为什么"少"反而更好

打个比方：你在 20 人的班级里选篮球队员。

- **技术面因子** = 看最近 20 天的训练表现（高度相关）
- **基本面因子（PE/PB/ROE）** = 看三个月前的体检报告（体脂率、肺活量）

问题不是体检报告没用——是"预测明天比赛谁能打好"这个任务里，体检报告的信息量已经被最近训练表现覆盖了。把体检数据加进评分公式，只是在干扰真正有用的信号。

这是今天最重要的洞察：**信号的时间尺度必须匹配预测窗口**。20 日的比赛评测窗口 = 只能看日频/周频数据。季频的财务数据在这个尺度上等于噪声。

## 除了砍因子，还做了什么

### 工程修复（v2.1，+1.6 分）

1. **数据源没装**：venv 里缺少 yfinance/akshare → 安装后从 0 分变正常
2. **数据源太慢**：yfinance 50 秒拉 3 只股票 → akshare 0.7 秒，加 baostock 当备胎
3. **全量数据塞给 AI**：49 只 × 250 天的价格矩阵全丢给 LLM → 改成内存缓存，AI 只看摘要
4. **Bull/Bear 没工具**：子 Agent 被限制了不能用量化工具 → 移除了 leader-only 限制

### 判市优化（v2.2，效果被 bug 抵消了）

1. **固定阈值改成波动率标准化**：原来"价格偏离均线 3% = 牛市"——高波市场和低波市场里 3% 完全不同。改成"偏离 N 个标准差"
2. **加 CSI 300 指数做第二参考**：个股等权均价噪音大，加了市值加权的指数做交叉验证

### 和 Gemini/Goone 的协作

今天花了大量时间在两个 AI 之间来回验证方案：

- **Goone 提出方向** → 我跑代码验证 → 数据说话
- 提出了 5 个改进方向，其中 3 个被数据驳回（指数加权、双视角评分、base-weight-swap）
- 核心价值不是 "谁对了"，而是 **"不被自己骗了"**——直觉说"多加点因子会更好"，数据说"加了反而更差"

## 79.7 是不是满分

不是。79.7/100。

- 收益端：40.3/56（72%）——还有空间但边际收益接近零
- 回撤端：20.4/24（85%）——接近硬上限
- 资源端：19.0/20（95%）

当前框架下（4 个价格因子 + 20 日窗口 + 49 只股票），79.7 近似天花板。要进一步需要新的信息源（情绪数据、高频数据、另类数据），或者调仓频率的改变——但这由比赛规则决定。

## 你该记住的关键词

1. **IC（信息系数）**：衡量因子和未来收益的统计关系。>0.1 才有用，<0.05 是噪声
2. **因子冗余**：两个因子相关系数 >0.7 → 它们说的是同一件事，只留一个
3. **时间尺度匹配**：20 日窗口只能用日频因子。季频数据是噪声
4. **数据验证 > 直觉**：每个改动都跑 scoring.py 看分数，不靠"听起来合理"
5. **双 AI 协作**：一个提方案，一个跑验证，互相质疑。比一个人闭门造车靠谱
