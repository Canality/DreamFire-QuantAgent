# 06 — 数据源与级联兜底

## 为什么需要多个数据源

量化策略的第一步是拿到股票数据。数据源不是"挑一个最好的用"，而是"准备好多个备用的，坏一个还有下一个"。

### 我们的三层架构

```
akshare (东方财富) → baostock (自营服务器) → yfinance (Yahoo Finance)
     ↓ 失败              ↓ 补充缺失               ↓ 最后兜底
```

每层只抓上一层没拿到的股票，不重复抓。

### 为什么 akshare 不够

akshare 的数据来自东方财富、新浪等国内财经网站。问题是：

- 这些网站对高频请求有限流——请求太快会被封 IP
- 公司内网/代理环境可能拦截对 eastmoney.com 的请求
- 实际测试：49 只股票中 akshare 只拿到了 0-5 只（公司网络代理拦截）

### 为什么加 baostock

BaoStock 有三个关键优势：

1. **自有服务器**：数据来自自己的数据库，不依赖爬虫。东方财富封了不影响它
2. **无频率限制**：不需要积分，不需要注册，不会封 IP
3. **A 股全覆盖**：历史数据可追溯到 1990 年，含 PE/PB/ROE 等基本面数据

实测：49 只股票 baostock 全部拉到，耗时 ~15 秒。

### 什么是级联兜底

```
def _fetch_real_data(tickers, start, end):
    all = {}
    
    # Tier 1: akshare (国内快)
    prices, errors = _fetch_akshare(tickers, start, end)
    all.update(prices)
    
    # Tier 2: baostock 补充缺失的
    missing = [t for t in tickers if t not in all]
    if missing:
        prices2, _ = _fetch_baostock(missing, start, end)
        all.update(prices2)
    
    # Tier 3: yfinance 兜底剩下的
    still_missing = [t for t in tickers if t not in all]
    if still_missing:
        prices3, _ = _fetch_yfinance(still_missing, start, end)
        all.update(prices3)
    
    return all  # 最终覆盖率 = 三层贡献之和
```

关键设计：**每一层只看上一层没拿到的**。这样 akshare 拿到 5 只 + baostock 拿 44 只 = 49 只，yfinance 不需要启动。

### 效果对比

| | 改之前 | 改之后 |
|------|------|------|
| 数据源 | yfinance 单源 | akshare → baostock → yfinance |
| 成功率 | 33/49 (首次) | 49/49 |
| 耗时 | 328s (yfinance 慢) | 15-60s |
| 数据缺失导致评分下降 | 73.1 | 78.5 |

## 和比赛的关系

比赛要求 Agent 在评测期内跑回测，评测期是未知的。如果评测期那几天 akshare 刚好被封了，yfinance 又连不上——你的 Agent 就会因为"拿不到数据"直接拿 0 分。多源兜底就是为了防止这种情况。

## 从中学到什么

1. **不信任单一外部依赖**：任何网络 API 都可能挂。准备至少两个独立的备选
2. **每层只补充缺失**：不要重新抓已有的——浪费时间和网络带宽
3. **baostock 是 A 股最佳选择**：自有服务器、无需注册、无频率限制、含基本面数据
