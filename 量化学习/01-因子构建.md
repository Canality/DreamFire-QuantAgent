# 01 — 因子构建：从 Feature Engineering 到 Alpha 信号

**写给有 ML 背景的人**：因子（Factor）就是金融领域的特征（Feature）。你在 Kaggle 里对原始数据做 feature engineering，量化研究员对行情数据做因子构建——本质上是一回事。区别只在于：ML 特征的 target 是 label，因子的 target 是**未来收益率**。

---

## 1. 因子是什么

> 因子 = 一个能解释或预测股票收益的量化指标

类比：你在 Kaggle 里构造了 `user_purchase_ratio_7d` 这个特征来预测用户会不会点击广告。在量化里，你构造 `momentum_20d` 来预测股票未来 20 天会不会涨。

把因子值排个序，最高分的那组股票未来表现应该好于最低分那组——如果确实如此，这个因子就"有效"。

---

## 2. 因子的经济学基础

因子不是随便拿价格算个数就叫因子。每个有效因子背后都有一个**经济学故事**——为什么这个数字能预测未来收益？

### 2.1 动量因子（Momentum）

**公式**：

$$ MOM_n = \frac{P_t}{P_{t-n}} - 1 $$

其中 $P_t$ 是今天的价格，$P_{t-n}$ 是 n 天前的价格。

**经济学故事**：趋势会延续。为什么？

- **行为金融解释**：投资者对新信息的反应不足（underreaction），好消息出来后股价慢慢涨，形成趋势
- **风险解释**：动量强的股票往往处于景气上行周期，基本面在改善
- **A 股特殊性**：散户占比高，追涨杀跌行为强化了动量效应

**ML 类比**：相当于你发现某用户最近 7 天点击频率持续上升，预测他明天还会继续点——行为惯性。

**参数选择**：
- 20 日动量（~1 个月）：捕捉短期情绪和资金流向
- 60 日动量（~1 个季度）：捕捉中期基本面趋势

> **Canaan 注意**：动量在趋势市（单边上涨或下跌）表现极好，在震荡市（横盘上下波动）表现极差。这是因子的"市场环境依赖"——后面讲到 IC 分析时会展开。

### 2.2 波动率因子（Volatility）

**公式**：

$$ \sigma_{ann} = std(R_{20d}) \times \sqrt{252} $$

其中 $R_{20d}$ 是近 20 个交易日的日收益率序列。

**经济学故事**：低波动股票长期跑赢高波动股票。

这违反了教科书上的 CAPM 理论（高风险 = 高收益），所以被称为**"低波动异象"（Low Volatility Anomaly）**。

为什么低波动反而好？
- **杠杆约束**：机构不能加杠杆买低波动股票，导致低波动股票被系统性低估
- **彩票偏好**：散户喜欢买高波动的"彩票股"，推高了它们的价格，压低了未来收益
- **行为偏差**：投资者过度关注高波动股票的上涨潜力，忽略了它们跌起来也更狠

**ML 类比**：模型 ensemble 里，你不会给方差最大的那个模型最高权重——相反，你会降低它的权重。风险平价的逻辑：波动大的股票在组合里的贡献要压低。

**我们的做法**：波动率因子取负值——波动率越低，得分越高。

### 2.3 最大回撤因子（Max Drawdown）

**公式**：

$$ MDD = \max_{t \in [0,T]} \frac{P_{peak}(t) - P(t)}{P_{peak}(t)} $$

翻译成人话：在最近 60 天里，从最高点跌到最低点，最大跌幅是多少。

**经济学故事**：回撤小的股票防御性强。当市场下跌时，这些股票跌得少，说明有资金在护盘或基本面足够扎实。

**为什么给 25% 最高权重？** 因为比赛评分里"最大回撤"占了 24 分。策略设计要**对齐评分标准**——这是 Kaggle 经验：知道 scoring metric 后，优化方向要跟 metric 一致。

**ML 类比**：相当于你在模型选择时不仅看 validation accuracy，还看它在各个 fold 上的稳定性。回撤大的股票 = 不同 fold 上 accuracy 波动大的模型——不可靠。

### 2.4 成交量趋势因子（Volume Trend）

**公式**：

$$ V_{ratio} = \frac{MA_5(V)}{MA_{20}(V)} $$

**经济学故事**：成交量突然放大意味着有资金关注。可能是机构在建仓（利好），也可能是主力在出货（利空）。单独看量不告诉你方向，但配合价格走势就很有用。

**ML 类比**：相当于你在特征里用了"用户最近活动频率 / 历史平均活动频率"——突然的活跃度变化通常是重要信号。

## 3. 行业中性化：你的 StandardScaler 加了分组

这是今天策略里最重要但最容易被忽略的技术细节。

### 为什么不能直接用原始因子值排名？

假设你有两个股票：

| 股票 | 板块 | 20日动量 |
|------|------|----------|
| 宁德时代 | 新能源 | +8% |
| 工商银行 | 金融 | +3% |

如果你直接按动量排名，宁德时代永远排在工商银行前面——不是因为它更好，而是因为新能源板块天然波动大。

这就像你在 Kaggle 里有两个国家的用户数据，一个国家的人均消费 $500，另一个 $50。你不做归一化，模型会认为"国籍"是最重要的特征。

### 行业中性化的做法

在每个板块**内部**做 Z-score：

$$ z_i^{金融} = \frac{x_i - \mu_{金融}}{\sigma_{金融}} $$

$$ z_j^{新能源} = \frac{x_j - \mu_{新能源}}{\sigma_{新能源}} $$

做完之后：
- 工商银行的动量 z-score 可能是 +1.5（远超银行板块平均）
- 宁德时代的可能是 +0.3（在新能源里表现普通）

结果：工商银行反而排在前面，因为它是**板块内的领先者**。

**ML 直接类比**：

```python
# 你在 Kaggle 里可能这样做
from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# 金融里的行业中性化 = Grouped StandardScaler
for sector in sectors:
    mask = (df['sector'] == sector)
    df.loc[mask, 'factor_z'] = (
        df.loc[mask, 'factor'] - df.loc[mask, 'factor'].mean()
    ) / df.loc[mask, 'factor'].std()
```

## 4. 因子权重：为什么这样分配？

我们策略的权重：

| 因子 | 权重 | 逻辑 |
|------|------|------|
| 20日动量 | 25% | 短期趋势，超额收益主要来源 |
| 60日动量 | 15% | 中期趋势，辅助验证短期信号 |
| 波动率 | 20% | 低波动溢价，风险控制 |
| 最大回撤 | 25% | **对齐比赛评分** |
| 成交量趋势 | 15% | 辅助信号，资金流向确认 |

权重的设计原则：
1. **风险因子合计 45%** → 回撤在比赛评分中占 30%，所以给了最高权重
2. **动量因子合计 40%** → 收益在评分中占 70%，所以也给了高权重
3. **这两者之间的张力很重要** ——高动量往往伴随高波动，多因子模型就是在"想涨"和"怕跌"之间找平衡

## 5. 从 Feature Engineering 到 Factor Research 的思维转换

| ML Feature Engineering | Quant Factor Research |
|------------------------|----------------------|
| 构造特征 → 训练模型 → 看 CV score | 构造因子 → 排序选股 → 看 IC |
| 特征重要性（feature importance） | IC（因子与收益的相关性） |
| 防止过拟合（regularization） | 样本外测试（out-of-sample） |
| 交叉验证（k-fold CV） | 分段回测（不同时间段验证） |
| 特征越多越好？→ 维度灾难 | 因子越多越好？→ 过拟合风险 |

你在 Kaggle 里可能习惯一个模型有 100+ 个特征。但在量化里，**10 个以内**的有效因子通常就够了——因子越多，过拟合风险越大（下一课讲 IC/IR 时会展开）。

## 6. 本节关键概念速查

| 概念 | 一句话 | ML 近似 |
|------|--------|---------|
| 因子 | 预测股票收益的量化指标 | Feature |
| 动量 | 趋势会延续 | 时间序列自相关 |
| 波动率 | 价格的不确定性 | 标准差 |
| 最大回撤 | 从峰值到谷底的最大跌幅 | worst-case validation gap |
| Sharpe | 单位风险的超额收益 | 归一化的 accuracy |
| 行业中性化 | 板块内 Z-score 标准化 | Grouped StandardScaler |
| IC | 因子与未来收益的相关性 | Feature importance |
