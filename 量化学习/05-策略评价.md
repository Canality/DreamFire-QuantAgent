# 05 — 策略评价体系：什么才算一个好策略？

**写给有 ML 背景的人**：模型好不好，不能只看一个 accuracy。策略好不好，不能只看收益率。你需要一个多维度的评价体系——就像你用 precision、recall、F1、ROC-AUC、confusion matrix 来全方位评估模型。

---

## 1. 策略评价的三层金字塔

```
         ┌──────────────┐
         │   稳健性      │  ← 最顶层：换个时间还灵吗？
         │   Robustness  │
         ├──────────────┤
         │   风险调整    │  ← 中间层：赚的钱值得承担的风险吗？
         │   Risk-Adj    │
         ├──────────────┤
         │   收益       │  ← 底层：赚了多少钱？
         │   Returns     │
         └──────────────┘
```

大多数新手只看第一层。Kaggle 老手会看第二层（因为你知道 CV 的重要性）。很少有人主动看第三层——而这就是业余和职业的分界线。

---

## 2. 第一层：收益指标

### 2.1 累计收益率

$$ R = \frac{V_{end} - V_{start}}{V_{start}} $$

最直观。比赛评分里占了 56 分（80×70%）。

### 2.2 超额收益（Alpha）

$$ \alpha = R_{portfolio} - R_{benchmark} $$

你的策略相对基准（如沪深 300）的超额收益。

两个策略：
- A：赚了 20%，沪深 300 赚了 25% → Alpha = -5%（不如买指数）
- B：赚了 15%，沪深 300 赚了 5% → Alpha = +10%（显著跑赢）

**策略 B 虽然绝对收益低，但策略能力更强。**

**ML 类比**：你的模型 accuracy 85%，baseline model 83%——真正属于你模型的贡献只有 2%。

---

## 3. 第二层：风险调整指标

### 3.1 Sharpe 比率

$$ Sharpe = \frac{R_{ann} - R_f}{\sigma_{ann}} $$

衡量"性价比"。已在前两篇详细讲过。

**Sharpe 的局限性**：假设收益服从正态分布。A 股有明显的负偏（negative skew）——暴跌比暴涨更频繁、更剧烈。Sharpe 对上行波动和下行波动一视同仁，但实际上只有下行波动才是你真正害怕的。

### 3.2 Sortino 比率（改进版 Sharpe）

$$ Sortino = \frac{R_{ann} - R_f}{\sigma_{downside}} $$

只看下行波动率（只算亏损日的标准差）。

同样是 10% 波动率的两个策略：
- A 的波动来自大涨（你不在乎）
- B 的波动来自大跌（你极其在乎）

Sharpe 认为 A 和 B 一样好，Sortino 认为 A 更好。

### 3.3 Calmar 比率

$$ Calmar = \frac{R_{ann}}{MDD} $$

**年化收益 / 最大回撤**。越高越好。

我们的策略：40.55% / 4.77% ≈ 8.5。意味着每承担 1% 回撤风险，获得 8.5% 的年化回报。

**ML 类比**：这就像你在两个模型之间选择——一个 accuracy 90%/worst fold 85%，另一个 accuracy 92%/worst fold 70%。你会选前者，因为它更可靠。

### 三个风险调整指标对比

| 指标 | 公式 | 关注点 | 适用场景 |
|------|------|--------|---------|
| Sharpe | R/σ | 总风险 | 通用 |
| Sortino | R/σ_down | 下行风险 | 非对称收益 |
| Calmar | R/MDD | 极端风险 | 基金评价 |

---

## 4. 第三层：稳健性检验

这是职业量化研究员和业余爱好者分道扬镳的地方。

### 4.1 样本外测试

把你的回测周期切成三段：

```
├── 训练期(2023)  │  开发策略、调因子权重
├── 验证期(2024)  │  测试策略、选择最佳参数
├── 测试期(2025)  │  只看一次！最终评价
```

你只能在训练期和验证期碰数据和调参数。测试期是神圣不可侵犯的——一旦你看了测试期结果然后回去调参数，它就"被污染"了。

**ML 精确类比**：train/val/test split 的铁律。你在 Kaggle 里一定知道不能拿 test set 调参——在量化里这条规则更重要，因为金融数据本身就少。

### 4.2 分段回测

在不同的子时间段上分别评估：

| 时间段 | 市场状态 | 策略收益 |
|--------|---------|---------|
| 2023 H1 | 震荡市 | +5.2% |
| 2023 H2 | 下跌市 | -2.1% |
| 2024 H1 | 上涨市 | +18.7% |
| 2024 H2 | 震荡市 | +6.4% |

一个好的策略：在各种市场状态下**都不差**（不一定是最好，但不能有致命缺陷）。

一个过拟合的策略：在训练期（2023）表现完美，在测试期（2025）崩盘。

### 4.3 敏感性分析

把因子权重上下浮动 ±20%，看结果变化有多大：

```python
# 原始权重
base_return = 18.87%

# 动量因子权重从 25% 调整到 20% 或 30%
perturbed_return_1 = 16.2%   # 变化 -2.67%
perturbed_return_2 = 20.1%   # 变化 +1.23%

# 波动范围在可接受范围内 → 策略对参数不敏感 → 稳健
```

如果微调一个参数导致收益率从 +18% 变成 -5%，说明策略极度不稳定——大概率过拟合。

### 4.4 换手率与交易成本

一个好的策略不能只在忽略成本时表现好。要考虑：

**换手率**：每次调仓有多少股票被换掉？
- 高换手（> 50%）：交易成本会大幅侵蚀收益
- 低换手（< 20%）：策略稳定，成本低

**我们的策略预期**：月频调仓，每次换手约 30-40%。

---

## 5. 综合策略评价清单

每轮实验后，逐项检查：

```
□ 累计收益率（比赛权重 56 分）
□ 最大回撤（比赛权重 24 分）
□ Sharpe 比率（> 1.0 可接受，> 2.0 优秀）
□ Calmar 比率（> 1.0 可接受）
□ 样本外测试通过（测试期不退化太多）
□ 多时间段表现一致（不是只有某一年好）
□ 参数敏感性低（±20% 参数变化 → 收益变化 < 30%）
□ 板块分散度（6 个板块至少覆盖 5 个）
□ 换手率合理（月换手 < 50%）
□ 能解释为什么有效（不是纯数据挖掘）
```

最后一条最重要。如果你不知道为什么策略有效，它大概率只是拟合了历史噪声。

---

## 6. 本节关键概念速查

| 概念 | 一句话 |
|------|--------|
| Alpha | 剥离市场涨跌后，你的策略真正的赚钱能力 |
| Sortino | 只看下行波动的 Sharpe（更严格） |
| Calmar | 年化收益 / 最大回撤（衡量极端风险性价比） |
| 样本外测试 | 留一段完全不看的数据，只看一次 |
| 分段回测 | 在牛/熊/震荡市分别评估策略 |
| 敏感性分析 | 微调参数看结果变化（变化大 = 过拟合） |
| 换手率 | 每次调仓换掉多少股票（影响交易成本） |
